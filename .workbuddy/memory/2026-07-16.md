# 2026-07-16

## 修复：启动告警 "Standard Commons Logging discovery in action with spring-jcl"
- 现象：应用启动日志出现 `please remove commons-logging.jar from classpath` 告警（spring-jcl 检测到 classpath 同时存在 commons-logging，两者都实现 JCL 冲突）。
- 根因：`platform-starter` 引入 `aliyun-sdk-oss:3.16.1` → `aliyun-java-sdk-core:4.5.10`，后者以 compile 作用域（非 optional/provided）传递引入 `commons-logging:1.2`；该 core 还经 `httpclient:4.5.12` 再次带入 commons-logging。
- 排查手段：本机 Maven 工具链在沙箱损坏（boot/plexus-classworlds jar 内无 Launcher 类、`java` 二进制异常），无法跑 `dependency:tree`；改用在本地仓库（`D:\Program Files\maven`）反查各 pom：commons-logging 在 Druid(provided+optional)、POI(optional)、mybatis(optional)、aliyun-java-sdk-core(compile) 等处的 scope 差异，确认仅 aliyun-java-sdk-core 为真实传递来源。其余 REAL 候选（腾讯云/微信/jjwt/老 httpclient4.5/ camunda 等）均不在本项目依赖树。
- 修复：在 `platform-starter/pom.xml` 的 `aliyun-sdk-oss`、`cos_api`、`minio` 三个云存储 SDK 依赖上加 `<exclusion>` 排除 `commons-logging`（排除会级联到 aliyun-sdk-oss 整个子树，含 httpclient→commons-logging 路径），统一保留 Spring 的 spring-jcl 作为 JCL 实现。
- 验证：用户本机 `mvn compile -pl platform-admin -am` 后启动，告警应消失。
