# 2026-07-18 工作日志

## 1. CPU 监控字段扩充（ServerBO.Cpu / ServerComponent / ServerVO）
- 背景：原 Cpu 只有 cpuNum + sys/used/free，但 sys==used（同源于 getSystemCpuLoadBetweenTicks），有效信息仅“逻辑核心数 + 总使用率”。
- 新增 4 字段：`cpuName`(型号) / `physicalNum`(物理核心) / `packageNum`(CPU 插槽) / `perCore`(List<Double> 每核心使用率)。
- 采集改动（ServerComponent.setCpuInfo）：
  - cpuName 取 `processor.getProcessorIdentifier().getName()`（OSHI 6.x 的 getName 在 ProcessorIdentifier 上，不在 CentralProcessor 直接，踩坑一次）。
  - 复用既有的 Util.sleep(1000) 采样窗口：sleep 前多采一次 `getProcessorCpuLoadTicks()`，sleep 后 `getProcessorCpuLoadBetweenTicks(prevPerCore)` 得每核心负载。
  - physicalNum=`getPhysicalProcessorCount()`，packageNum=`getPhysicalPackageCount()`。
- ServerVO.Cpu 同步加同名字段，Orika 按名自动映射，无需额外配置。
- 注意：loadAverage（getSystemLoadAverage）未加，因 Windows 上 OSHI 返回 -1，环境是 Windows 无价值。
- 验证：admin `mvn -o -pl platform-admin -am compile` BUILD SUCCESS（修正 getName 编译错误后）。

## 2. 服务器信息(Sys)扩充 4 字段
- 新增：`systemUptime`(系统已运行时长, 复用 getRunTime()) / `bootTime`(启动时间戳 epoch ms) / `computerModel`(硬件厂商+型号, 来自 ComputerSystem) / `processCount`(系统进程总数)。
- 采集改动（ServerComponent.setSysInfo 改为接收 SystemInfo si）：
  - `os.getSystemUptime()`(秒)→ 转 startTime 喂 getRunTime 得格式化时长；`os.getSystemBootTime()`(秒)×1000 → bootTime。
  - `si.getHardware().getComputerSystem()` 取 manufacturer+model 拼 computerModel，单独 try 防异常。
  - `os.getProcessCount()` → processCount。
  - 新增 import：oshi.software.os.OperatingSystem、oshi.hardware.ComputerSystem。
- ServerBO.Sys / ServerVO.Sys 同步加同名字段，Orika 按名自动映射。
- 验证：admin `mvn -o -pl platform-admin -am compile` BUILD SUCCESS（仅 unchecked/deprecation 提示，无错误）。
