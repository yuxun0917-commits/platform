# 2026-07-19 工作日志

## 1. 登录/二次认证接入 RSA 非对称加密（后端）
- 背景：用户决定「HTTPS 由网关终结、后端只做 RSA 应用层兜底」。密码存储层 BCrypt 不变，传输层加 RSA。
- 新增文件（platform-starter/.../security）：
  - `RsaComponent.java`：构造时生成 RSA 2048 密钥对，私钥仅驻留内存（不落盘/不进仓库/不打日志）；`getPublicKeyBase64()` 返回 X.509 DER 的 Base64；`decryptByPrivateKey(String)` 用 `RSA/ECB/PKCS1Padding` 解密（与前端 jsencrypt 默认算法一致）。使用 java.util.Base64，无额外依赖。
  - `RsaAutoConfiguration.java`：`@Bean RsaComponent`（@ConditionalOnMissingBean），注册进 `META-INF/spring/.../AutoConfiguration.imports`（新增第 7 个自动配置）。
- 改动（platform-admin）：
  - `vo/LoginVO.java`、`vo/SecondAuthVO.java`：各加 `Boolean encrypted = false`（默认 false，保证 Swagger/curl 明文调用仍可用）。
  - `controller/AuthController.java`：注入 `RsaComponent`；新增 `GET /auth/rsa/public-key` 返回公钥 Base64；`login()` 与 `secondAuth()` 在 encrypted=true 时用私钥解密后再走 BCrypt matches。
- 验证：`mvn -o -pl platform-admin -am compile` BUILD SUCCESS。
- 关键约束（告知前端）：① 公钥接口返回的是裸 Base64，jsencrypt 需先包 `-----BEGIN PUBLIC KEY-----` / `-----END PUBLIC KEY-----` PEM 头尾才能 setPublicKey；② 每次启动密钥对重建，前端必须「登录前现取公钥」再加密，不能写死；③ 分别加密 username、password（各 ≤245 字节，够用）；④ 仅 RSA 无 HTTPS 时 token 仍明文，本方案依赖网关 HTTPS 兜底。
- 用户安全决策（记 MEMORY.md）：本项目传输安全 = 网关 HTTPS（终结 TLS，后端零配置）+ 后端 RSA 应用层兜底；密码存储 BCrypt 不变。

## 2. 移除 encrypted 判断，改为无条件加解密
- 用户决定：前端必须用 RSA 加密、后端无条件解密，不需要 `encrypted` 标志位判断。
- `LoginVO` / `SecondAuthVO` 删除 `Boolean encrypted` 字段。
- `AuthController.login()`：`username/password` 直接 `rsaComponent.decryptByPrivateKey(...)`（不再包 if）。
- `AuthController.secondAuth()`：`password` 直接解密。
- 副作用：Swagger/curl 明文登录不再可用（解密会失败），符合「强制加密」预期。`GET /auth/rsa/public-key` 仍保留供前端取公钥。
